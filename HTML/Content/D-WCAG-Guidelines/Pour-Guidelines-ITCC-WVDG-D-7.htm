WCAG POUR

The four POUR principles, which established the foundation that enables anyone to access and use Web content, organized the guidelines and success criteria of web accessibility. The four principles are Perceivable, Operable, Understandable, and Robust.

  • POUR Accessibility: It breaks accessibility into clear steps to support inclusive design and sets guidelines and tips to make your web content more accessible to your users.

  • POUR Inclusivity: Using these principles helps designers and developers make content that works for all users, regardless of ability.

P - Perceivable

Information and user interface components must be presented in a manner that is perceivable to all users.

That is, users must be able to perceive the content being delivered; it must not be invincible to all of their senses.

O – Operable

User interface components and navigation must be fully operable by users.

This implies that users must be capable of interacting with the interface; it should not rely on interactions that are beyond their ability to perform.

U – Understandable

Both the information provided and the operation of the user interface must be understandable.

In other words, users must be able to comprehend the content and how to use the interface; neither should be so complex as to be inaccessible to the user’s understanding.

R – Robust

Content must be developed to be robust enough to be reliably interpreted by a wide range of user agents, including assistive technologies.

How can your web content fit the four POUR principles?

To achieve perceivability:

  • Use Alternative Text–

Providing alternative text for your images and charts can help those who cannot see your visuals, but rely on a screen reader. Alternative text allows screen readers to describe visuals to users who are visually impaired.

  • High Contrast–

Use significant color contrast between text and background to improve readability for users with low vision and to make your web content more readable for all users.

  • Captions and Transcripts–

Add captions to videos and transcripts to audio content to assist users who are hard of hearing or deaf.

  • Adjustable Text–

Enable text resizing and page zooming without loss of content or functionality, to accommodate users with varying visual requirements.

To achieve operability:

  • Clear Navigation–

Establish a logical and consistent navigation structure that allows users to locate and move between content sections with ease.

  • Keyboard Accessibility–

Ensure that all interactive elements, such as buttons and links, are fully operable via keyboard alone, to support users who are unable to use a mouse.

  • Sufficient Time–

Refrain from imposing strict time limits on user tasks, or provide options to extend the allotted time, thereby accommodating users who require additional time to complete actions.

  • Avoid Flashing Content–

Minimize or eliminate flashing and blinking content to reduce the risk of inducing seizures in users with photosensitive conditions.

To achieve understanding:

  • Use plain language–

Keep wording simple and avoid technical terms so everyone can understand the content.

  • Consistent Navigation–

Use the same layout and menu structure on all pages to help users know where to find things.

  • Help with errors–

Show clear messages and tips when users make mistakes in forms, so they know how to fix them.

  • Readable Text–

Use clear fonts, enough spacing, and organized headings to improve readability.

To achieve robustness:

  • Semantic HTML–

Structure your content with proper HTML tags so screen readers and other tools can understand it.

  • ARIA attributes–

Use ARIA roles and labels to help screen readers work better with interactive or changing content.

  • Compatibility Testing–

Check your content on different devices and with assistive tools to make sure everything works well.

  • Keep code error-free–

Write valid, error-free code to avoid problems that could affect accessibility.
